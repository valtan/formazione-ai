# context/

Cartella dei **file di contesto stabili**. Claude (Cowork) li legge all'inizio
di ogni task per inquadrare il lavoro.

## File presenti

- `profilo-formatore.md` — Chi sono, come lavoro, approccio didattico
- `glossario-it.md` — Terminologia italiana uniforme per AI/ML
- `normative-riferimento.md` — Sintesi GDPR, AI Act, normative italiane
- `architettura-progetto.md` — Mappa e filosofia della struttura della cartella
- `ente-organizzatore-piamarta.md` — Identità, sedi, contatti e accreditamenti A.F.G.P.
- `linee-guida-formazione-base.md` — Struttura e principi del percorso formativo
- `contesto_questionario_assesment.md` — Logica del questionario di assessment
- `brand-style-guide.md` — **Guida di stile vincolante** (colori, font, layout, logo, tono) per ogni produzione visiva o di comunicazione
- `logo_piamarta_formazione.png` — Logo ufficiale dell'ente (regole d'uso nella guida di stile)

## Regola per le produzioni visive

Ogni produzione con elementi visivi o di comunicazione (brochure, slide,
volantini, PDF, social, presentazioni) deve **sempre** applicare
`brand-style-guide.md` e le regole d'uso del logo.

## Quando aggiornare

- Quando cambia il mio modo di lavorare
- Quando aggiungo terminologia nuova al glossario
- Quando entra in vigore una normativa nuova
- Quando evolve l'identità visiva (aggiornare `brand-style-guide.md`, non crearne copie)
